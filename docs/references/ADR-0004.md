# ADR-0004: kage のポート方針を固定ポートに限定（動的ポートの棄却）

- **Status**: Accepted (2026-06-29)
- **関連**: 基本設計書 §5-7（結論の所在）／ ADR-0001（衝突時 stop の親基準）／ §10-1（整合先）／ §3.3（ゾンビ非識別の行き先）

## Context

kage MCP サーバが listen するポート番号を、**固定ポート**（実装時に番号を確定）にするか、**動的ポート**（serve 起動時に OS から空きポートを取得）にするかを決める必要があった。

決定を縛る前提：kage は `init` が各面（WSL / Windows の各 Claude Code）の MCP 設定へ、サーバの**エンドポイントを静的な文字列として書き込む**（クロスOS配線）。`serve` とは別タイミングで、各面の設定ファイルに番号が焼き付く。この「静的に書いて、後から serve する」構造がポート方針を縛る。

## Decision

**固定ポート**を採用する。番号は実装時に確定する。動的ポートは採らない。

## 棄却した代替案：動的ポート

- **実在する利点**：起動のたびに OS から空きを取るため、ポート衝突を**自動回避**できる。「ポートが埋まっていて起動できない」が原理的に起きない。
- **棄却根拠（クロスOS追従問題）**：動的ポートでは serve のたびに番号が変わる。しかし各面の MCP 設定には `init` が**静的な番号**を焼き付けている。番号が毎回変われば、設定は次の serve で即座に陳腐化し、各面が古い番号を指して到達不能になる。これを追わせるには「serve のたびに全面の設定を書き換える追従機構」が必要で、WSL↔Windows の設定ファイルを横断して毎回それを行うのは高コストかつ脆い。固定ポートは「設定に書く文字列が常に同じ」ことで、この問題を**構造的に消す**。
- **コスト面**：固定は OS から空きを取得するロジックも不要で、動的より実装が軽い。

→ 動的ポートの「衝突を自動回避できる」という利点は、*静的にクロスOS配線する* kage の設計の下では「追従問題」という重い欠点に裏返る。トレードオフは固定に傾く。

## 受容するリスクと、その帰結としての衝突時の態度

固定ポートは衝突を防げない。対象ポートが他プロセスに占有されていれば起動できない。この不便さは**意図的に受容**する（§3.3 相当の割り切り）。

受容する以上、「占有されていたらどう振る舞うか」を決めねばならない。この振る舞いは新たに基準を立てず、**ADR-0001（self-heal vs stop の2軸：①正解の一意性 ②他者非侵襲）のインスタンス**として決まる。self-heal の候補は2つあり、両方とも2軸で落ちる：

- **別ポートへ自動フォールバック** → **①違反**。各面の設定は固定ポートを指したままなので、別番号で起動しても誰も到達できない。「直すべき終状態」が一意でないどころか、本決定が消した追従問題を蘇らせる。
- **占有プロセスを kill** → **②違反**。占有者は他者が意図して建てたプロセスでありうる。非侵襲を破る。

両 self-heal ルートが塞がるため **stop**。占有プロセスの調査コマンドと停止をユーザーへ案内して終了する。これは ADR-0001 の個別適用であり、§10-1「silent failure を避ける／原因を明示」と整合する。

## Consequences

- 正典 §5-7 には、固定ポート採用・追従問題による棄却の結論・受容リスク・衝突時 stop の結論（ADR-0001 準拠）を最小記述で残す。動的ポート棄却の詳細トレードオフと、衝突 self-heal の2ルートが2軸で落ちる導出は本 ADR が保持する。
- **ゾンビプロセス非識別は本 ADR の射程外**。「占有者が前回の kage 自身か他プロセスかを PID で識別するか」は、ポート方針の精緻化ではなく**衝突診断の深さに関する独立した機能スコープ判断**であり、Arcana 第3「やらないことを決める」の素直な適用（希少 × 低被害 × 高コスト）にすぎない。よって ADR の deliberation ではなく、**§3.3「やらないこと」の conclusion** として記す。
- 衝突時 stop と ADR-0001 を紐付けたことで、ポート以外の占有・衝突系の振る舞いも同じ2軸で読めるようになる。

---

> **正典側の対応**：§5-7 を上記「残す」項目の最小形へ刈り込み、ゾンビ非識別の行を §3.3 へ新設する。
