# ADR-0001：エラー時の self-heal / stop 分岐基準

- **Status**: Accepted (2026-06-29)
- **関連**: 基本設計書 §10-1（不変条件の結論）／ §5-7・§7-3・§9-2（個別適用）

## Context

kage の全操作（init / serve / status / uninstall）に共通する「失敗時にどう振る舞うか」の横断方針が必要だった。当初はエラーをケースごとに列挙して個別対応を決めかけたが、それでは横断的な判断基準にならず、新規ケースのたびに再判断が要ると判断し、方針を「一貫した規準を1つ立てる」へ転換した。

## Decision

self-heal（黙って直す）か stop（止めて観測者へ委ねる）かを、次の **2軸の AND** で分岐する。

1. 正解が一意に確定する（直すべき終状態が曖昧でない）
2. 他者の領域に非侵襲（kage 自身の痕跡の範囲に閉じ、他者が意図して設定した状態を侵さない）

両方が成立するときに**のみ** self-heal。どちらか一方でも欠ければ stop。

中核の敵は silent failure（無言×害）＝失敗しているのに告げず利用者が成功と誤認する状態であり、この分岐はそれを避けるための規準である。

## 棄却した代替案

### 案1：内部要因 / 外部要因で分岐（例：外部要因なら止まる）

- **反例**：§5-7 ポート衝突は「外部要因」だが、止まるのが正しい。
- **反例**：孤児マーカー（§9-2）は「内部要因」だが、直す（self-heal）のが正しい。
- 内部/外部という軸では正しく振り分けられず、棄却。

### 案2：他者への影響の有無「だけ」で分岐

- **反例**：`ABOUTME.md` が空/不在のように、kage 自身の領域内（他者非侵襲）で起きる失敗でも、正解が一意でないため止めるべき。
- 他者影響の軸だけでは「①正解の一意性」を問えず、誤って self-heal を許す。一意性の軸が欠けるため、単独軸としては不十分で棄却。

### 2軸の必要性（なぜ AND か）

- 案1・案2はいずれも単一軸では破れる。
- 純粋な②ケース（read-only な設定ファイルへの書き込み）が AND の必要性を示す好例：終状態は一意（①満たす）だが、到達には他者が意図して設定した read-only 領域への侵襲（chmod）が要る（②落ちる）。よって止まる。①が満たされても②が欠ければ止まる、という関係が OR ではなく AND であることを裏づける。

## Consequences

- 新しいエラーが現れても、この2軸に当てれば結論が出る。判断が場当たりにならない。
- 「どう外したか」の思考ログは本 ADR に封じ込め、正典（§10-1）には結論（判断基準）のみを置く。汎用原則は Arcana.md へ昇格する。
- 個別適用（§5-7 ポート衝突 stop、§7-3 三状態明示、§9-2 孤児 self-heal）は、すべてこの不変条件のインスタンスとして読める。
