# ADR-0005: 面の検出を `.claude/` ディレクトリ存在で判定（コマンド検出の棄却）

- **Status**: Accepted (2026-06-29)
- **関連**: §9-1（結論の所在）／ ADR-0001（先回り作成＝self-heal の親基準）／ ADR-0002（(c) と (b) の収束）／ §10-1（整合先）

## Context

`kage init` は各面（WSL / Windows の各 Claude Code）を「検出」してから配線する。ここでの検出が答えるべき問いを、まず正しく定義する必要があった。

検出の目的は **Claude Code を起動できるか**ではない。kage は Claude Code を起動しない。kage が知りたいのは「**自分が書き込む先**（`.claude/` ディレクトリと設定ファイル）が存在するか／書き込めるか」だけである。この目的のズレを取り違えると、検出方法を誤る。

guiding concern は一点：**書き込める面を「面なし」と取りこぼさない（false negative を出さない）**こと。

## Decision

面の検出は **`.claude/` ディレクトリの存在**（書き込み先そのもの）で判定する。`.claude/` が無ければ kage が先回りで作成する（後述、ADR-0001 のインスタンス）。

## 棄却した代替案：コマンド検出（`claude` 実行コマンドの有無）

「`claude` コマンドが PATH 上にあるか」で面の有無を判定する案。一見、Claude Code の存在を確かめる素直な方法に見える。

- **棄却根拠（false negative）**：「インストール済みだが PATH が未通過」というケースで、`.claude/` には**書き込めるのに**コマンドが見つからず「面なし」と誤判定する。これは guiding concern（書ける面を取りこぼさない）に正面から反する。
- **根本原因**：コマンド検出が測っているのは「**実行能力**（起動できるか）」であって、kage が本当に必要とする「**書き込み先の有無**」ではない。Context で定義した目的のズレが、そのまま誤検出として表面化する。

→ 検出は「実行能力」ではなく「書き込み先の存在」を直接見るべきで、それは `.claude/` ディレクトリの存在確認に他ならない。

## 採用した自己修復：`.claude/` 不在時の先回り作成（ADR-0001 のインスタンス）

`.claude/` 存在で判定すると、「`.claude/` の生成タイミング（インストール時か初回起動時か）」に検出が依存しうる、という不確実性が残る。これを **kage が先回りで作る**ことで吸収する。

この「黙って作る」振る舞いは新たな基準を立てず、**ADR-0001（self-heal vs stop の2軸）の self-heal 側のインスタンス**として正当化される：

- **①正解の一意性**：取るべき終状態は「`.claude/` が在る」一通りに確定する。
- **②他者非侵襲**：`.claude/` は Claude Code 標準の無害なディレクトリであり、他者が意図して設定した状態を侵さない。

両軸が立つため self-heal してよい。作成した事実は `registry.json` に `isCreated: true` で記録し、アンインストール時の undo 対象とする。

> **foreclosed**：対立案「`.claude/` の事前存在を要求し、無ければ面なしとする」は、先回り作成を採用した時点で自動消滅する。生成タイミング依存の false negative を招くため独立した吟味の対象にもならない。棄却理由が*この決定の中にある*のではなく、上流（self-heal 採用）にあるため、本 ADR では棄却案として立てない。

## Consequences

- 正典 §9-1 には、検出目的（書き込み可否であって起動能力ではない）・`.claude/` 存在判定・不在時の先回り作成（`isCreated: true`）・3面の検出先パスを最小記述で残す。コマンド検出を false negative で退ける導出は本 ADR が保持する。
- 検出を「書き込み先の存在」に固定したことで、(c) Desktop【Code】タブ Local が (b) Windows Terminal と同じ `.claude/` へ収束する事実（ADR-0002）とも整合する——検出は「どの実行環境か」ではなく「どこへ書くか」を見るため、面が収束しても破綻しない。
- 先回り作成を ADR-0001 の self-heal インスタンスとして紐付けたことで、衝突 stop（ADR-0004）と合わせ、「黙って直す／止める」の両側がともに ADR-0001 の2軸から読めるようになった。

> **正典側の対応**：§9-1 を上記「残す」項目の最小形へ刈り込む（差分は別途提示）。
